# 绿联 NAS Nginx 403 Forbidden 错误故障排除

当您在绿联 NAS 上部署 Nginx 并遇到 **403 Forbidden** 错误时，通常是由于以下两个原因之一造成的：**文件权限不足** 或 **挂载路径错误**。

请按照以下步骤逐一排查。

## 原因一：文件权限不足 (最常见)

Nginx 容器内的运行用户可能没有权限读取您在 NAS 上上传的文件。

### 解决方法

1.  **打开绿联文件管理器**。
2.  找到您上传网站文件的文件夹（例如 `Docker/student-display`）。
3.  **右键点击**该文件夹，选择 **属性** 或 **权限** (根据系统版本可能略有不同)。
4.  确保 **所有用户 (Everyone)** 或 **其他用户 (Others)** 拥有 **读取 (Read)** 权限。
    *   如果可以，勾选 "应用到子文件夹和文件"。
5.  **重启 Docker 容器**。

---

## 原因二：挂载路径错误

如果您挂载的路径不正确，Nginx 在容器内的默认目录找不到 `index.html`，也会报 403 错误（因为禁止列出目录）。

### 解决方法

1.  打开绿联 Docker 应用。
2.  停止并编辑您的 `student-display` 容器。
3.  检查 **存储空间** 挂载设置：
    *   **错误示例**：
        *   主机路径：`/Docker/student-display` (父文件夹)
        *   装载路径：`/usr/share/nginx/html`
        *   *后果*：容器内结构变为 `/usr/share/nginx/html/dist/index.html`，Nginx 找不到根目录的 index.html。
    *   **正确设置**：
        *   主机路径：`/Docker/student-display/dist` (必须指向包含 index.html 的那个文件夹)
        *   装载路径：`/usr/share/nginx/html`
4.  保存并启动容器。

---

## 原因三：缺少 index.html

确认您的 `dist` 文件夹上传完整。

### 解决方法

1.  进入 NAS 文件管理器，打开 `dist` 文件夹。
2.  确认里面包含 `index.html` 文件。
3.  如果为空或缺少文件，请重新上传。

---

## 进阶方案：特权模式

如果以上都不行，可以尝试赋予容器更高权限。

1.  编辑容器。
2.  在 **基本设置** 或 **能力** 选项卡中。
3.  勾选 **特权模式 (Privileged)** 或 **高权限执行**。
4.  重启容器。
